# Weka 高性能文件系统技术规格

## 硬件配置

### 固定配置
- **CPU**: Intel 5418Y
- **内存**: 32GB
- **NVMe SSD**: 4/8/12 块可选
- **SSD 容量**: 7.68TB 或 15.36TB
- **网络**: 100Gb×2 或 200Gb×2

### 最小配置要求
- **最少节点数**: 6 节点

## 容量计算

### 基础参数
- **TB 到 TiB 转换系数**: 0.909
- **容量保留系数**: 0.9（10% 用于元数据和系统开销）

### 保护方案（纠删码）

| 节点数范围 | 数据块 (D) | 校验块 (P) | 条带宽度 (D+P) | 可用容量比例 |
|-----------|-----------|-------------|-------------|
| 6-9       | 5         | 2-3       | 7-8           | D/(D+P)     |
| 10-99     | 8       | 2-3       | 10-11         | D/(D+P)     |
| 100+      | 8         | 4     | 12            | D/(D+P)     |

**约束条件**:
- D > P（数据块必须大于校验块）
- 5 ≤ D+P ≤ 20（条带宽度范围）
- 默认保护级别为 N+2

### 容量公式
```
可用容量 (TiB) = 节点数 × NVMe数 × SSD容量(TB) × 0.909 × (D/(D+P)) × 0.9
```

**示例**:
- 6 节点 × 8 块 NVMe × 15.36TB × 0.909 × (5/7) × 0.9 = 339.5 TiB

## 性能计算

### 性能公式系数

基于文档数据推导的性能系数：

| 指标 | 每盘每节点系数 | 单位 |
|------|--------------|------|
| 写带宽 | 0.6625 | GB/s |
| 写 IOPS | 27,000 | IOPS |
| 读 IOPS | 225,000 | IOPS |
| 读带宽（无瓶颈） | 4.346 | GB/s |

### 网络带宽限制

| 网络类型 | 每节点理论带宽 | 实际可用带宽 |
|---------|--------------|----------|
| 100Gb×2 | 25 GB/s      | 22.5 GB/s   |
| 200Gb×2 | 50 GB/s      | 45 GB/s     |

### 性能计算公式

```javascript
// 写性能（不受网络限制）
写带宽 (GB/s) = 节点数 × NVMe数 × 0.6625
写 IOPS = 节点数 × NVMe数 × 27,000

// 读性能（受网络带宽限制）
读 IOPS = 节点数 × NVMe数 × 225,000
读带宽 (GB/s) = min(节点数 × NVMe数 × 4.346, 节点数 × 网络带宽每节点)
```

### 公式验证

**6 节点，100Gb×2，8 NVMe**:
- 计算读带宽 = min(6×8×4.346, 6×22.5) = min(208.6, 135) = 135.0 ✓
- 计算写带宽 = 6×8×0.6625 = 31.8 ≈ 31.9 ✓
- 计算读 IOPS = 6×8×225000 = 10,800,000 ✓
- 计算写 IOPS = 6×8×27000 = 1,296,000 ✓

**6 节点，200Gb×2，8 NVMe**:
- 计算读带宽 = min(6×8×4.346, 6×45) = min(208.6, 270) = 208.6 ≈ 208.7 ✓
- 计算写带宽 = 6×8×0.6625 = 31.8 ≈ 31.9 ✓

**8 节点，100Gb×2，8 NVMe**:
- 计算读带宽 = min(8×8×4.346, 8×22.5) = min(278.1, 180) = 180.0 ✓
- 计算写带宽 = 8×8×0.6625 = 42.4 ≈ 50.6 ❌

注意：8 节点数据的写带宽与公式有偏差，可能文档数据有误或使用了不同的保护方案系数。实际实现中使用统一公式。

## 配置优化策略

在满足容量和性能需求的前提下，优先选择：
1. **最少节点数**（降低硬件成本和管理复杂度）
2. **最少 NVMe 数量**（降低单节点成本）
3. **最小 SSD 容量**（7.68TB 优先于 15.36TB）

## 使用场景

- **高性能计算 (HPC)**: 大规模并行 I/O
- **AI/ML 训练**: 高吞吐量数据加载
- **媒体处理**: 4K/8K 视频编辑
- **数据库**: 高 IOPS 事务处理
- **大数据分析**: 高带宽顺序读写
