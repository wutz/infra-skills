# Weka 高性能文件系统规划工具

帮助用户根据容量和性能需求，规划 Weka 高性能文件系统部署方案，计算所需节点数量、选择合适的纠删码保护方案、确定最优 NVMe SSD 配置。

## 工作流程

此 skill 使用 JavaScript 脚本 `scripts/weka-planner.js` 进行核心计算，流程如下：

1. 收集用户需求（容量、性能、网络类型、保护级别）
2. 调用 `weka-planner.js` 脚本进行计算
3. 解析并格式化输出结果

## 用户输入需求

### 1. 容量需求（必填）

询问用户的容量需求，支持十进制和二进制单位：
- **十进制单位**：TB、PB（1000 进制）
- **二进制单位**：TiB、PiB（1024 进制）—— **推荐使用**

始终向用户推荐使用二进制单位（TiB、PiB），因为它们与实际存储计算更匹配。

### 2. 性能需求（可选）

询问用户是否有性能需求：
- **读带宽**：例如 200 GB/s、100 GiB/s
- **写带宽**：例如 50 GB/s、25 GiB/s
- **读 IOPS**：例如 10000000
- **写 IOPS**：例如 1000000

带宽单位支持十进制（MB/s、GB/s）和二进制（MiB/s、GiB/s）。如果用户未提供性能需求，输出时优先使用十进制单位。

### 3. 网络类型（可选）

- **200gb**（默认）：200Gb 双口×2 网卡（每节点 4×200Gb，总带宽 800Gb）—— **默认使用更高规格**
- **100gb**：100Gb 双口×2 网卡（每节点 4×100Gb，总带宽 400Gb）

网络类型会影响读带宽上限：
- 100Gb 双口×2：每节点最大 45 GB/s
- 200Gb 双口×2：每节点最大 90 GB/s

### 4. 保护级别（可选）

- **N+2**（默认）：容忍 2 个节点离线
- **N+3**：容忍 3 个节点离线
- **N+4**：容忍 4 个节点离线（100+ 节点推荐）

保护方案根据节点数自动选择：
- 6-9 节点：EC 5+P（P=2/3）
- 10-99 节点：EC 8+P（P=2/3/4）
- 100+ 节点：EC 8+4（推荐 N+4）

## 使用方法

### 步骤 1：收集用户需求
询问用户以下信息：
1. **容量需求**（必填）：例如 "500TiB"、"1.5PiB"
2. **性能需求**（可选）：
   - 读带宽：例如 "200GB/s"、"100GiB/s"
   - 写带宽：例如 "50GB/s"、"25GiB/s"
   - 读 IOPS：例如 "10000000"
   - 写 IOPS：例如 "1000000"
3. **网络类型**（可选）：100gb 或 200gb
4. **保护级别**（可选）：2、3 或 4

### 步骤 2：调用计算脚本

使用 Bash 工具调用 `scripts/weka-planner.js` 脚本：

```bash
cd /Users/wutz/Projects/wutz/infra-skills/skills/weka-planner
node scripts/weka-planner.js --capacity "500TiB" [--read-bw "200GB/s"] [--write-bw "50GB/s"] [--read-iops "10000000"] [--write-iops "1000000"] [--network-type 200gb] [--protection-level 2] --json
```

参数说明：
- `--capacity`：容量需求（必填）
- `--read-bw`：读带宽需求（可选）
- `--write-bw`：写带宽需求（可选）
- `--read-iops`：读 IOPS 需求（可选）
- `--write-iops`：写 IOPS 需求（可选）
- `--network-type`：网络类型（可选，默认 200gb）
- `--protection-level`：保护级别（可选，默认 2）
- `--json`：以 JSON 格式输出结果

### 步骤 3：解析并展示结果

脚本返回 JSON 数组，每个元素代表一个方案，包含：
- `recommended`：是否为推荐方案（第一个元素为 `true`）
- `configuration`：配置方案（节点数、CPU、内存、NVMe配置、网络、保护方案）
- `capacity`：可用容量和存储效率
- `performance`：性能指标（读/写带宽和 IOPS）
- `pros`：该方案的优点列表
- `cons`：该方案的缺点列表

所有方案合并为一张对比表格，每行为一个方案（推荐方案排第一行，标注"★ 推荐"），每列为一个指标，最后一列合并优点和缺点（格式：优：.../ 缺：...）。不包含存储效率列。不要在输出末尾添加"说明"、注释或补充信息。

## 硬件配置说明

### 固定配置
- **CPU**: Intel 5418Y
- **内存**: 32GB
- **NVMe SSD**: 4/8/12 块可选
- **SSD 容量**: 7.68TB 或 15.36TB
- **网络**: 100Gb 双口×2 或 200Gb 双口×2（每节点 2 块双口网卡）

### 最小配置要求
- **最少节点数**: 6 节点

## 性能特性

### 写性能（不受网络限制）
- 写带宽：每盘每节点 0.6625 GB/s
- 写 IOPS：每盘每节点 27,000 IOPS

### 读性能（受网络带宽限制）
- 读 IOPS：每盘每节点 225,000 IOPS
- 读带宽：受网络类型限制
  - 100Gb 双口×2：最大 45 GB/s/节点
  - 200Gb 双口×2：最大 90 GB/s/节点

## 重要说明

- 脚本仅支持 7.68TB 和 15.36TB NVMe SSD 规格
- 所有节点数的性能均使用计算公式（基于文档数据推导）
- 容量保留系数 0.9（10% 用于元数据和系统开销）
- 保护方案根据节点数和保护级别自动选择
- 输出单位与用户输入单位保持一致（二进制或十进制）
- 读带宽可能受网络带宽限制，写带宽和 IOPS 不受网络限制
- 配置优化策略：最少节点数 → 最少 NVMe 数量 → 最小 SSD 容量
- 默认配置 1 台热备节点：热备节点不参与容量计算，但参与性能计算
- 默认使用 200Gb×2 高规格网络；仅当用户明确指定时才使用 100Gb×2
