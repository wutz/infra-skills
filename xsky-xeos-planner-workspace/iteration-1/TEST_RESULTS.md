## 测试场景 1: small-capacity-no-perf

**输入需求：**
- 容量：500 TiB
- 性能：无特殊要求

**计算结果：**
- 服务器台数：3 台（强制最小值，EC4+2 要求）
- 纠删码方案：EC4+2（因为台数 < 5 台，无法使用 EC8+2）
- 磁盘配置：32 × 16TB HDD
- 可用容量：754.00 TiB
- 性能预估：
  - 上传带宽：2880 MiB/s (3.02 GB/s)
  - 下载带宽：5760 MiB/s (6.04 GB/s)
  - 上传 OPS：9,600 IOPS
  - 下载 OPS：28,800 IOPS

**验证结果：** ✓ 通过
- ✓ 容量满足需求（754 TiB ≥ 500 TiB）
- ✓ 正确选择 EC4+2（台数不足 5 台）
- ✓ 正确应用 EC4+2 最小 3 台服务器要求
- ✓ 计算过程清晰，展示了降级逻辑和最小台数检查

## 测试场景 2: large-capacity-high-bandwidth

**输入需求：**
- 容量：2 PB（十进制）
- 上传带宽：≥ 10 GB/s
- 下载带宽：≥ 20 GB/s

**计算结果：**
- 服务器台数：10 台
- 纠删码方案：EC8+2
- 磁盘配置：32 × 10TB HDD（从 16TB 降级到 10TB 以满足性能）
- 可用容量：1884.90 TiB (2.07 PB)
- 性能预估：
  - 上传带宽：9600 MiB/s (10.07 GB/s) ✓
  - 下载带宽：19200 MiB/s (20.13 GB/s) ✓
  - 上传 OPS：32,000 IOPS
  - 下载 OPS：96,000 IOPS

**验证结果：** ✓ 通过
- ✓ 正确转换单位（2 PB → 1818 TiB）
- ✓ 尝试多种磁盘大小（16TB → 12TB → 10TB）
- ✓ 最终配置满足所有性能需求
- ✓ 使用 EC8+2 提高空间效率

## 测试场景 3: small-capacity-high-ops

**输入需求：**
- 容量：80 TiB
- 上传 OPS：≥ 50,000 IOPS
- 下载 OPS：≥ 150,000 IOPS
- 上传带宽：≥ 2 Gbps
- 下载带宽：≥ 4 Gbps

**计算结果：**
- 服务器台数：16 台
- 纠删码方案：EC8+2
- 磁盘配置：32 × 16TB HDD
- 可用容量：4825.35 TiB（约 60 倍需求）
- 性能预估：
  - 上传带宽：15,360 MiB/s (128.86 Gbps) ✓
  - 下载带宽：30,720 MiB/s (257.72 Gbps) ✓
  - 上传 OPS：51,200 IOPS ✓
  - 下载 OPS：153,600 IOPS ✓

**验证结果：** ✓ 通过
- ✓ 识别出 OPS 需求主导配置（需要 500 块磁盘）
- ✓ 正确计算服务器台数（500 / 32 = 16 台）
- ✓ 所有性能指标均满足需求
- ✓ 说明了容量富余的原因（性能驱动）
